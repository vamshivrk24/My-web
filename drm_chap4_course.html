Chapter 4

PROCEDURES AND CURSORS


Procedure:

    A named group of SQL statements that have been pre-created and stored in the server database, Stored procedures accept input parameters. Each parameter has three parts: parameter name, direction, and data type.

    Syntax :

    CREATE PROCEDURE procedure_name (parameter 1, parameter 2, ..) IS/AS
    BEGIN SQL statement
    [EXCEPTION Exception_section]
    END;

    To Execute or call a stored procedure:
      1) EXEC procedure_name ();
      2) EXECUTE procedure_name ();
      3) CALL procedure_name ();
Cursor:

    In Oracle, a cursor is a pointer to a memory location on the database server that the DBMS uses to process a SQL query. In PL/SQL programs, you can use cursors to retrieve and manipulate data. There are two types of cursors: implicit and explicit

  • Implicit cursor is used, the Database Management System (DBMS) will perform the open, fetch and close operations automatically. Implicit cursors should be used only with SQL statements that return a single row.
    • Syntax:

    DECLARE CURSOR Cursor_name IS SELECT * FROM Table_Name
    WHERE Column_name=xyz;

  • Explicit cursors are queries defined using a name. An explicit cursor can be thought of as a pointer to a set of records and the pointer can be moved forward within the set of records. Explicit cursors provide the user the complete control over opening, closing and fetching data. Also, multiple rows can be fetched using an explicit cursor.
    • Syntax:

    OPEN cursor_name;
    FETCH cursor_name INTO record_name;
    CLOSE cursor_name;

  • The variables must be declared before being used (tablename.columnname%TYPE )
  • %TYPE data type is often used to display a single column value.
  • %ROWTYPE is used to display multiple column values.
