Data Resource Management:

Chapter 5

PACKAGES AND TRIGGERS


Package:

    A package is a group of related procedures and functions, togather with the cursors and variables they use, stored together in the database for continued use as a unit. A package is compiled and stored in the database. A package is created as package specification and package body.

    Syntax:

    CREATE PACKAGE name AS specification (visible part)
    public type and object declarations subprogram specifications
    END [name];

    CREATE PACKAGE BODY name AS body (hidden part)
    private type and object declarations subprogram bodies
    [BEGIN initialization statements]
    END [name];
Trigger:

    A trigger is a named program unit that is stored in the database and fired (executed) in response to a specified event. The specified event is associated with either a table, a view, a schema, or the database, and it is one of the following:

  • A database manipulation (DML) statement (DELETE, INSERT, or UPDATE)
  • A database definition (DDL) statement (CREATE, ALTER, or DROP)
  • A database operation (SERVERERROR, LOGON, LOGOFF, STARTUP, or SHUTDOWN)
  • You can use :OLD to refer to the values in the row before change and
  • :NEW to refer to the values in the changed row.
    • You can only use these two attributes within PL/SQL code.

    Syntax:

    CREATE TRIGGER trigger_name
    {BEFORE | AFTER} {INSERT | DELETE | UPDATE [of ]}
    ON table_name [FOR EACH ROW] | [FOR EACH STATEMENT]
