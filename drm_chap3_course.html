PL/SQL BLOCKS


The PL stands for Procedural Language. It is used by Oracle client software to manipulate database data. A PL/SQL program combines SQL queries with procedure commands for tasks such as manipulating variable values, evaluating IF/THEN decision control structures, and creating loop structures that repeat instructions. PL/SQL interfaces directly with Oracle database and can be stored directly in the database.

PL/SQL programs can be executed in Oracle SQL*Plus, SQL Developer or Forms Builder. PL/SQL is an interpreted language, which means that a program called the PL/SQL interpreter checks each command for syntax errors, translates each command into machine language, and then execute each command, one command at a time. With PL/SQL, an entire block of statements can be sent to Oracle at one time.

PL/SQL programs are structured in blocks and generally contain three sections: declaration, body and exception.

Syntax :

In a PL/SQL program block, the DECLARE and EXCEPTION sections are optional. The declare part is where variables are declared. All used variables must be declared in this section. The exception section is where we place error handling code. The body part is where your programs shall be placed. Here, you can have IF statements, loops, etc. The end signifies the end of this program block.

    Scalar Variables: variables with single value such as number, date or character.
