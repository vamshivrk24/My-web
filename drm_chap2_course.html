Data Resource Management:

CHAPTER 2

ADVANCED DATABASE OBJECTS

    VIEWS - VIRTUAL TABLES

  • A view is a virtual table based on a SELECT query
    • Syntax: Create VIEW name AS SELECT query
    UPDATABLE VIEW

  • Criterion to determine if a VIEW can update a base table:
  • In the view's output, if the primary key columns of the base table still have unique values, the base table is updatable. Use the syntax below.
    • Syntax :
    SELECT COLUMN_NAME, UPDATABLE, INSERTABLE, DELETABLE FROM USER_UPDATABLE_COLUMNS WHERE TABLE_NAME = 'PSVUPD';
    SYNONYMS

  • An alternative name for objects such as tables, views, sequences, and stored procedures. To create a synonym, the user needs to have the CREATE SYNONYM privilege.
  • Public and Private
    • Syntax : CREATE [public] SYNONYM [schema].synonym_name FOR [schema].object_name
    SEQUENCES

  • Generally used to create unique, sequential integer values that are used as primary key values in database tables.
    • Syntax : CREATE SEQUENCE [schema].sequence_name
    INDEXES

  • To improve data retrieval speed, Stores a map of column values and the ROWIDs of matching table rows.
  • B-tree index, Bitmap index, Function-based index , Partitioned index
    • Syntax : CREATE INDEX index_name ON table_name (column1, column2, . column_n)
